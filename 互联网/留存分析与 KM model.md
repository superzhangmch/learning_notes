
# KM / 留存分析 数据构造简要笔记

## 1. KM 数据格式

```text
(time, event)
event = 1 → 事件发生（流失 / 未续费）
event = 0 → 删失（观察结束时还没发生）
```

```text
(x,1) → 在 x 时发生事件
(x,0) → 至少到 x 时事件还没发生
```

note: 只有两种event, 没第三种

---

# 2. (x,0) 的核心原则（最重要）

```text
x = 最后一次确认“事件还没发生”的时间
```

不是猜的, 不是未来推测, 必须有证据（登录、活跃、会员有效期、签到、合同到期日等）

一句话总结：

```text
(x,0) = last known event-free time
```

# 3. 关于 x 要不要尽量大

正确说法是：

```text
只要有真实观测或证据支持，
(x,0) 的 x 越晚越好 → KM 越准确
```

本质原因：

```text
删失越晚 → 风险集信息越完整 → KM 更准确
```

但：

```text
可以晚观测，不能瞎延长
```

一句总结：

```text
可以观测得更晚，不能假设他活得更久
```

note: 也就是, 只要有途径获得可信的最晚存活时间, 就要用更晚的, 这样 KM 更准确. 比如会员续费问题, 如果今天做 KM 建模, 有的用户昨天才付费月会员, 若不考虑退费, 则未来一月他都是会员, 所以 x 可以取一月后.


----

# 4. KM 数据构造通用公式

（这个很重要，基本所有 survival 数据都这样做）

```text
time  = min(流失时间, 最后观测时间)
event = "流失" 如果流失在观察期内发生，否则 "存活"
```
