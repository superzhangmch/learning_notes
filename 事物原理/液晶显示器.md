CRT只有一个电子枪(或者说rgb 三个), 需要逐行扫描. 对于 LCD/OLED 来说, 也是逐行扫描显示的.

### 为啥液晶需要逐行显示
因为, LCD/OLED 的像素矩阵采用的是 行列矩阵寻址.

```
列驱动器 (Source Driver)
    ↓  ↓  ↓  ↓  ↓
→ [像素][像素][像素][像素]  ← 行1
→ [像素][像素][像素][像素]  ← 行2
→ [像素][像素][像素][像素]  ← 行3
↑
行驱动器 (Gate Driver)
每次只激活一行
```

每个像素背后有一个开关, Gate 线控制"开哪一行", Source 线同时写入该行所有像素的数据, 一次只能打开一行的 TFT，否则数据会串扰. 

对 1080p = 1920×1080 分辨率, 如果每个像素独立寻址 → 需要 207万 条独立控制线! 行扫描: 只需要 1920 + 1080 ≈ 3000 条线.

note:
- 电子纸 (E-ink)：可以全局刷新或局部刷新，不严格逐行. 理论上 OLED 每个像素是独立灯泡, 可以做到随机像素刷新，但驱动 IC 设计成本极高，没有必要
- 相机感光 CCD / CMOS: 由于同样的原因(没办法同时把几百万个像素的值都输出出去 —— 没有那么多引脚/导线), 也是逐行扫描的. 

### 为啥必须一直刷新, 即使静态图

- LCD: TFT 关闭后，像素电容保持电压, 但是电容会漏电！
- OLED 是电流驱动发光，不刷新就不发光，更需要持续驱动

而 E-ink 不需要.

### 屏闪

1. LCD 的频闪: 来源：背光亮度调节（PWM调光）
```
背光亮度100%：
|████████████████| 一直亮

背光亮度50%（PWM）：
|████░░░░████░░░░| 亮/灭交替
     ↑
  频闪！
```

2. OLED 的频闪: 来源：像素本身就是光源，PWM调光更严重

一个像素, 被行扫描点亮后, 持续发光, 哪怕扫描已经到了下一行. 如果用户调低亮度, 就需要这个灯珠亮度降低. 如果是调节电流法, 低亮度时：电流太小, 色彩容易漂移, 均匀性变差. 于是采用另一种方式, 让灯珠在亮灭之间交替. 单位时间进入人眼的光也是表少, 让你感觉暗了. 这就是 oled 的频闪. 但是, 人眼其实错觉为很暗, 同时又间歇接收到的是高亮, 部分人的视觉系统对这种波动比较敏感，因此会感到眼睛不适. 

屏幕越暗, 频闪越严重. 大于某个亮度, 用调节电流法, 不再有频闪问题.
