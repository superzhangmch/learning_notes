现在加密算法的主流套路是, 先用一种比较重的方法协商出密钥(RSA/ECC椭圆/量子法), 拿到密钥后用轻量的 AES 等对于内容做加密.

### RSA

RSA 也可以用于对内容做加密, 但是效率低.

### ECC椭圆曲线加密算法

椭圆ECC: 其实是对于有限域上的椭圆曲线来做的. 一般椭圆曲线(y² = x³ + a·x + b)确实是连续曲线, 而有限域上的椭圆曲线( y² ≡ x³ + a·x + b (mod p) ), 其实不再是曲线, 甚至不是离散曲线, 而是整体上平均分布散步在 x-y 图里的. 看下图:

**椭圆曲线上的加法**: 在曲线上取两点P, Q, 则会有第三个交点, 取这个交点的关于x-axis 的对称点作为 P+Q. 于是定义了该曲线点集合{ (x, y) } 上的加法. 如果是P+P, 则去切线的另一个交点来取对称获得 2P. 数学上可以证明, 这确实是一个加法. 如下: 

<img width="1574" height="692" alt="image" src="https://github.com/user-attachments/assets/7723a21b-2753-48f9-8b53-6910cb19aea4" />

<img width="1602" height="736" alt="image" src="https://github.com/user-attachments/assets/1a59c108-3ca1-4417-bc05-63f7ff350359" />

**有限域上**: 选定一个椭圆上的点 G, 按上面方法, 就可以迭代 k 次, 并求出 kG=G+G+.. + G. 对于大的k, 是可以用加法分配律用二分法快速算出的. 如图: 可以看到, { kG } 确实是随机均匀分布的.

<img width="1610" height="1000" alt="image" src="https://github.com/user-attachments/assets/eb63187a-f511-4dd3-a3a4-c715a9d05c01" />

**实际如何用于加密**: 实际中, 曲线方程, 与 G 都是公开的 & shared, 所有人都用一个. 自己选一个很大的迭代次数 k, 然后把 kG 结果公开, 别人拿到 kG, 没法反解出 k. 这个 k 就是密钥, 可以用于内容加密.
