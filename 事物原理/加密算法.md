现在加密算法的主流套路是, 先用一种比较重的方法协商出密钥(RSA/ECC椭圆/量子法), 拿到密钥后用轻量的 AES 等对于内容做加密.

### RSA

RSA 也可以用于对内容做加密, 但是效率低. 基于大数分解, 两个大数乘积是公钥, 自己留下的因子是私钥匙. 他扛不住未来的量子 Shor 算法破解(可高效做因式分解), 而 ECC 也怕.
```
密钥怎么生成(核心五步)

① 选两个大质数     p, q
② 算模数          n = p · q                    ← 公开
③ 算欧拉函数       φ(n) = (p−1)(q−1)            ← 保密
④ 选公钥指数       e, 要求 1<e<φ 且 gcd(e,φ)=1   ← 公开(常用 65537)
⑤ 算私钥指数       d = e⁻¹ mod φ(n)             ← 绝密
                  (即 e·d ≡ 1 mod φ)

- 公钥 = (n, e) —— 公开给所有人
- 私钥 = d —— 只有你有(p、q、φ 也要一起销毁/保密)
```

### ECC椭圆曲线加密算法

一句话说, 也是一种公钥+私钥的加密算法.

椭圆ECC: 其实是对于有限域上的椭圆曲线来做的. 一般椭圆曲线(y² = x³ + a·x + b)确实是连续曲线, 而有限域上的椭圆曲线( y² ≡ x³ + a·x + b (mod p) ), 其实不再是曲线, 甚至不是离散曲线, 而是整体上平均分布散步在 x-y 图里的. 看下图:

**椭圆曲线上的加法**: 在曲线上取两点P, Q, 则会有第三个交点, 取这个交点的关于x-axis 的对称点作为 P+Q. 于是定义了该曲线点集合{ (x, y) } 上的加法. 如果是P+P, 则去切线的另一个交点来取对称获得 2P. 数学上可以证明, 这确实是一个加法. 如下:

**有限域上**: 选定一个椭圆上的点 G, 按上面方法, 就可以迭代 k 次, 并求出 kG=G+G+.. + G. 对于大的k, 是可以用加法分配律用二分法快速算出的. 如图: 可以看到, { kG } 确实是随机均匀分布的.

**实际如何用于加密**: 实际中, 曲线方程, 与 G 都是公开的 & shared, 所有人都用一个(怎么从这么复杂的曲线中选一个它上面的离散点 G, 也是有方法的). 自己选一个很大的迭代次数 k(是类似 1E99423A4ED27608A15A2616A2B0E9E52CED330AC530EDCC32C8FFC6A526AEDD 这么大. 256bit. 但是用倍加法, 也就256次迭代而已. 其实很快), 然后把 kG 结果公开, 别人拿到 kG, 没法反解出 k. 这个 k 就是密钥. k 是私钥，kG 是公钥，但要产生"共享密钥"还需要一个协议。最常见的是： ECDH（椭圆曲线 Diffie-Hellman）密钥交换：
```
Alice 私钥: a,  公钥: aG
Bob   私钥: b,  公钥: bG

Alice 计算: a·(bG) = abG  ✅
Bob   计算: b·(aG) = abG  ✅
```
双方得到同一个点 abG → 派生出 AES 密钥
窃听者只有 aG 和 bG，无法得到 abG（这就是椭圆曲线离散对数难题）。
