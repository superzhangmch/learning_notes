# Mobile Attribution / MMP / SRN / S2S — QA 笔记

## Q1：AppsFlyer / Adjust 是干嘛的？

**答：**
它们是 **MMP（Mobile Measurement Partner）**，主要做三件事：

1. 记录广告点击 / 展现
2. 记录 App 安装 / 事件
3. 把安装 / 收入归因到广告渠道

一句话总结：

> **MMP = 归因系统 + 广告数据对账系统**

本质是一个：

```text
Click / Impression log
+ Install / Event log
→ Attribution Engine
```

基于 MMP 的归因, 整体流程与依赖各方: 

```
                ┌──────────────┐
                │ Ad Network   │
                │ Facebook     │
                │ Google       │
                └──────┬───────┘
                       │ click
                       ↓
                ┌──────────────┐
                │   MMP        │
                │ AppsFlyer    │
                │ Adjust       │
                │ Click DB     │
                │ Install DB   │
                └──────┬───────┘
                       ↑
                       │ install/event
                ┌──────┴───────┐
                │   App SDK    │
                │ 你的 App      │
                └──────────────┘
```

# Q2：归因（Attribution）到底在干嘛？

归因引擎主要做三件事：

```text
1. Device matching（是不是同一个设备）
2. Attribution window（时间窗口）
3. Last click rule（最后点击归因）
```

简化逻辑：

```text
Install happens
→ 找匹配的 click
→ 判断是否在 window 内
→ 取最后一次 click
→ 归因
```

伪代码：

```sql
SELECT last_click
FROM clicks
WHERE device_match
AND click_time < install_time
AND time_diff < window
ORDER BY click_time DESC
LIMIT 1
```

如果没有 click：

```text
→ 看 impression
→ 否则 organic
```

# Q3：MMP 怎么知道有人看了广告（impression）？

MMP 自己**看不到广告展现**，impression 数据一定来自广告平台。

来源有三种：

| 来源                             | 说明                      |
| ------------------------------ | ----------------------- |
| Ad Network impression postback | 广告平台发给 MMP              |
| SRN API 查询                     | 安装时向 Facebook/Google 查询 |
| SKAdNetwork                    | Apple 返回                |
| Tracking pixel                 | Web                     |

非常重要的一句话：

> **MMP 不拥有 impression 数据，impression 数据来自广告平台。**

---

# Q4：Facebook / Google 会把数据给 AppsFlyer 吗？

会，但不是全部数据。

广告平台不会给：

* 用户级 raw impression log
* 用户级 click log
* 用户 ID
* 用户画像

通常只会给：

```text
这个 install 是否来自我的广告
campaign = xxx
adset = xxx
click / impression
```

也就是：

> **他们共享的是归因匹配结果，不是完整广告日志。**

---

# Q5：这些数据是不是只给 MMP，不给 App？

大多数情况下可以这么理解：

| 数据                    | App | MMP |
| --------------------- | --- | --- |
| Aggregated report     | ✔   | ✔   |
| Install / event       | ✔   | ✔   |
| Click-level           | 有时  | ✔   |
| Impression-level      | 很少  | ✔   |
| Attribution match API | ✖   | ✔   |

原因：

1. 隐私
2. 安全
3. 防作弊
4. 接口统一
5. 平台生态控制
6. MMP 是认证 partner

行业结构是：

```text
Ad Network → MMP → App
```

不是：

```text
Ad Network → 每个 App
```

# Q6：什么是 SRN？

SRN = Self Reporting Network
例如：

| SRN              |
| ---------------- |
| Facebook         |
| Google Ads       |
| Apple Search Ads |
| TikTok           |
| Snapchat         |

SRN 的特点：

* 不走 tracking link
* 不给 click log
* 安装后通过 API 查询归因
* 归因由广告平台决定

流程：

```text
Install → MMP → Ask Facebook → Facebook return attribution
```

---

# Q7：什么是 S2S？和 SRN 有关系吗？

没有直接关系。

**S2S = Server-to-Server event reporting（事件上报方式）**
**SRN = Attribution method（归因方式）**

一句话总结：

```text
S2S = data how to send
SRN = who decides attribution
```

四种组合都可能存在：

```text
Tracking link + SDK
Tracking link + S2S
SRN + SDK
SRN + S2S
```

---

# Q8：做了 S2S 之后，还需要 MMP 吗？

大多数情况下 **仍然需要**。

因为 S2S 只解决：

```text
event tracking
purchase
revenue
server events
anti fraud
```

但 MMP 还负责：

```text
install attribution
SRN integration
SKAdNetwork
deep link
postback
fraud detection
ROI / cohort
multi-network attribution
```

只有当公司自己实现下面这些系统，才能不用 MMP：

```text
Click tracking
Install tracking
Attribution engine
Facebook SRN API
Google Ads API
SKAdNetwork
Postback system
Fraud detection
Deep link
Reporting
```

这基本等于自己做一个 AppsFlyer。

# Q9：为什么 Facebook / Google 不自己做 MMP？

核心原因：

```text
1. 广告平台不能既卖广告又当归因裁判（利益冲突）
2. 广告主会投多个平台，需要统一归因
3. 广告主不想把所有数据给广告平台
4. 需要一个 measurement layer
5. Measurement 会增加广告支出
```

行业结构是三方博弈：

```text
Ad Network
    ▲
    │
    │ attribution / postback
    │
MMP
    ▲
    │ SDK / S2S
    │
App / Advertiser
```

# Q10：整个移动广告归因系统的本质是什么？

这个总结非常重要：

```text
Ad Network 拥有：
    impression
    click
    cost

App 拥有：
    install
    revenue
    user behavior

MMP 做的事情：
    把两边的数据连接起来
    决定归因
```

一句行业总结：

```text
Ad networks know who saw the ad.
Apps know who paid money.
MMP connects the two.
```

## Q11：Web 网页应用是怎么归因的？不用 MMP 吗？

**答：**
Web 归因一般不用 AppsFlyer / Adjust，而是用：

| Web Analytics / Attribution |
| --------------------------- |
| Google Analytics (GA4)      |
| Adobe Analytics             |
| Mixpanel                    |
| Amplitude                   |
| Segment                     |
| Snowplow                    |

Web 归因核心技术：

```text
UTM parameters + Cookies + Pixel tracking
```

也就是说：

```text
Click URL → 带 UTM
→ 浏览器写 cookie
→ 用户注册 / 购买
→ 根据 cookie 归因
```

典型 URL：

```text
https://example.com/?utm_source=google&utm_campaign=spring&utm_medium=cpc
```

网站把这些参数写入 cookie：

```text
cookie:
utm_source = google
utm_campaign = spring
```

之后用户购买时：

```text
conversion → attributed to google / spring campaign
```

所以：

> **Web Attribution 基于 Cookie 和 URL 参数。**

# Q12：Web 归因流程

典型 Web 广告 → 转化流程：

```text id="web_attr_flow"
Ad (Google / Facebook)
    ↓
Click with UTM
    ↓
Landing Page
    ↓
Write Cookie
    ↓
User signup / purchase
    ↓
Analytics reads cookie
    ↓
Attribution
```

Web 归因用到的技术：

| 技术           | 用途     |
| ------------ | ------ |
| UTM 参数       | 标记广告来源 |
| Cookie       | 识别用户   |
| Pixel        | 记录转化   |
| Referrer     | 来源网站   |
| Session      | 会话     |
| LocalStorage | 存储信息   |

# Q13：为什么 App 归因比 Web 复杂很多？

这是一个非常重要的理解点。

| Web       | App        |
| --------- | ---------- |
| Cookie    | 没有 Cookie  |
| URL       | 没 URL      |
| 浏览器       | App 沙盒     |
| Referrer  | 没 referrer |
| JS        | SDK        |
| Page view | App open   |
| Pixel     | SDK event  |
| 用户 ID     | Device ID  |
| 登录        | Install    |

所以 Web 可以用：

```text
cookie → identify user
```

App 只能用：

```text
device id → match install
```

所以 Mobile Attribution 变成一个独立行业（MMP）。

一句非常经典总结：

```text
Web attribution = cookies
Mobile attribution = device IDs + install matching
```

# Q14：Web 和 App 归因技术对比

这个表很重要：

| Web              | Mobile App         |
| ---------------- | ------------------ |
| Cookie           | IDFA / GAID / OAID |
| UTM              | Tracking link      |
| Pixel            | SDK                |
| Page view        | App open           |
| Conversion pixel | In-app event       |
| Referrer         | Install referrer   |
| Analytics        | MMP                |
| Session          | Install            |
| Retargeting      | Re-engagement      |
| Web analytics    | Mobile attribution |

可以理解为：

```text
Google Analytics ≈ AppsFlyer（在各自世界里）
```
