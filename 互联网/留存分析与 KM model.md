
# KM / 留存分析 数据构造

### 1. KM 数据格式

```text
(time, event)
event = 1 → 事件发生（流失 / 未续费）
event = 0 → 删失（观察结束时还没发生）
```

```text
(x,1) → 在 x 时发生事件
(x,0) → 至少到 x 时事件还没发生
```

note: 只有两种event, 没第三种


### 2. (x,0) 的核心原则（最重要）

```text
x = 最后一次确认“事件还没发生”的时间
```

不是猜的, 不是未来推测, 必须有证据（登录、活跃、会员有效期、签到、合同到期日等）

一句话总结：

```text
(x,0) = last known event-free time
```

### 3. 关于 x 要不要尽量大

正确说法是：

```text
只要有真实观测或证据支持，
(x,0) 的 x 越晚越好 → KM 越准确
```

本质原因：

```text
删失越晚 → 风险集信息越完整 → KM 更准确
```

但：

```text
可以晚观测，不能瞎延长
```

一句总结：

```text
可以观测得更晚，不能假设他活得更久
```

note: 也就是, 只要有途径获得可信的最晚存活时间, 就要用更晚的, 这样 KM 更准确. 比如会员续费问题, 如果今天做 KM 建模, 有的用户昨天才付费月会员, 若不考虑退费, 则未来一月他都是会员, 所以 x 可以取一月后.


### 4. KM 数据构造通用公式

（这个很重要，基本所有 survival 数据都这样做）

```text
time  = min(流失时间, 最后观测时间)
event = "流失" 如果流失在观察期内发生，否则 "存活"
```

# KM（Kaplan–Meier）模型的输出

- 它的本质上只有一个东西: `生存函数 S(t)`
- 也就是： `S(t) = 在时间 t 之后仍然没有发生事件的概率`
- 如果是在留存分析里： `S(t) = 用户在 time t 之后仍然留存 / 仍然是会员 的概率`

### 1. KM 输出的东西

KM 模型通常会输出一条表或一条曲线，类似这样：

| time | survival S(t) |
| ---- | ------------- |
| 0    | 1.0           |
| 5    | 0.9           |
| 10   | 0.8           |
| 20   | 0.65          |
| 30   | 0.50          |

- 意思是： `S(10) = 0.8 → 有 80% 的人活过 10`
- 在留存里就是： `10 天后还有 80% 用户没流失`
- 所以： `KM curve = 留存曲线`

### 2. KM 具体怎么算（核心公式）

KM 是在每个事件时间更新一次：

$$
S(t) = \prod_{t_i \le t} \left(1 - \frac{d_i}{n_i}\right)
$$

其中：

* n_i = 在时间 t_i 前还在风险集的人数
* d_i = 在时间 t_i 发生事件的人数

可以理解为：

```text
每个时间点的存活率 = (这一刻没出事的人 / 这一刻还在观察的人)
总生存率 = 一直乘起来
```

# KM 一些理论

比如元器件寿命留存, 或者互联网会员留存等, 有不同场景. 他们的随时间的表现也不一样, 对于寿命, 时间越久, 越容易流失, 对于会员留存, 则相反, 越久越不容易流失. 所以KM model 只是一类(不是一种模型分布，而是一种估计方法（non-parametric estimator）), 而不是只是一个具体model. 

甚至过程可以连续, 可以离散. 但是最好理解的是月会员续费这样的离散情况, 下面我们借助于它.

第x月留存的人, 在第 x+1 月仍有流失. 那么随着 x 的取值的不同, 这个流失率会怎样变? 无非三种: 恒定、递增、递减. 对于互联网按月自动扣款, 往往是递减的, 因为留下的人更不容易取消. 

### hazard 

反映以上这个流失率的, 在 KM 分析中叫 hazard, 代表瞬时流失率. 

| 模型                 | hazard 形状  | 留存曲线   | 常见场景     |
| ------------------ | ---------- | ------ | -------- |
| Exponential        | 常数         | 指数下降   | 最简单模型    |
| Weibull            | 单调增 / 单调减  | 灵活     | 用户留存最常用  |
| Log-normal         | 先升后降       | 中间流失最多 | 行为类      |
| Log-logistic       | 先升后降（尾部更重） | 长尾     | App / 游戏 |
| Gompertz           | 指数上升       | 后期加速流失 | 老化/设备    |
| Gamma              | 可变         | 较灵活    | 异质人群     |
| Mixture / Cure     | 最终 plateau | 有长期用户  | 订阅 / 社区  |
| Piecewise constant | 分段常数       | 工业界常用  | 留存/LTV   |

最常见的其实是：Weibull、Log-normal、Log-logistic、Cure model、Piecewise hazard, 而不是假定流失率不随时间变化的 Exponential。

所有生存模型，本质都是在假设 hazard(t) 的形状。

### 从 hazard 角度理解所有模型（核心理解）

记住一句话：

> **所有生存模型，本质都是在假设 hazard(t) 的形状。**

不同模型 = 不同的 hazard 曲线。

#### 1. Exponential

[
h(t) = \lambda
]
常数

```
hazard
|
|-------
|
+------ time
```

---

#### 2. Weibull

[
h(t) = k \lambda^k t^{k-1}
]

```
k<1: 下降
k=1: 常数
k>1: 上升
```

非常灵活，所以最常用。

---

#### 3. Log-normal

hazard 形状：

```
hazard
|
|   /\
|  /  \
| /    \
+-------- time
```

解释：

* 刚开始不流失（新用户观察期）
* 用一段时间后开始流失
* 留到很久的人又很稳定

很多产品符合这种模式。

---

#### 4. Log-logistic

和 log-normal 类似，但尾巴更长（long tail），
在游戏 / 社区 / 内容产品里很常见。

---

#### 5. Gompertz

[
h(t) = \lambda e^{\gamma t}
]

hazard 指数上升：

```
hazard
|
|    /
|   /
|  /
| /
|/
+------ time
```

常用于：

* 人类死亡率
* 设备老化
* 电池寿命
* 保险精算

---

#### 6. Cure / Mixture model（非常重要）

这个在互联网留存里特别常见。

假设：

> 有一部分用户永远不会流失

模型：

[
S(t) = p + (1-p)S_0(t)
]

其中：

* p = 永久留存用户
* S₀(t) = 会流失的人群 survival

曲线：

```
留存
1.0 |\
    | \
    |  \
    |   \
    |    \______
    |            ---- plateau
    +-----------------------
```

很多产品最后都有 plateau：

* 会员
* 社区
* 工具软件
* SaaS
* 游戏
* 银行卡
* 手机号
* 微信
* Slack
* Notion
